# WeightNest 项目规则

## ⛔ 铁律

### 1. 禁止擅自改方案
遇到任何障碍（包冲突、编译失败、环境问题等），必须先：
- 列出 2-3 个可行选项
- 等小豆选择
- 再动手

严禁自己替小豆做决定。

### 2. 禁止擅自改依赖
修改 pubspec.yaml 依赖前必须先征求确认。

### 3. 禁止擅自改架构
大的架构变更（如 PG→SQLite）必须征求确认。

### 4. 修改完代码必须 commit + push
每次代码变更后立即提交推送，不留未推送的改动。

### 5. 每阶段完成后确认
完成一个阶段后必须先向小豆确认是否进入下一阶段。

### 6. 单个 session 只处理一个功能
超过 20 轮必须新建 session。

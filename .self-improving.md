# WeightNest 开发教训

## 代码生成后必做
- drift: `grep "^class " database.g.dart` 确认数据类名，不要假设
- 检查 `insertReturningId` 等 API 是否存在（drift 2.22 vs 2.28 不同）

## 算法方向
- 涉及排序/方向的数据，在注释写死 "最新在前" 或 "升序"
- 写完算法用一个手工案例纸面验证一遍

## UI 组件
- DropdownButtonFormField 不能放Dialog/BottomSheet里 → 用点击+BottomSheet列表替代
- 写完新screen立刻 flutter analyze，commit 前再 analyze

## 异步
- Riverpod: `.read().whenData()` ≠ "等待数据后执行" → 用 `.read().future.then()`
- 首次加载 vs 缓存：区分清楚 provider 的两种状态

## CSS
- 大段 CSS 修改后用工具数括号: `node -e "..."` 验证嵌套闭合
- @media 缺失闭合 → 所有断点失效

## 提交习惯
- 每步 commit，不要攒到最后一个大 commit
- 新文件创建后立刻 analyze
- 改完代码 → analyze → build → 确认 APK 文件名含版本号
